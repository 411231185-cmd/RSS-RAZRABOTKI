# Навык: catalog_etalon_check

## Назначение
Проверка описания товара (станок/деталь) на соответствие эталонному стандарту ТД РУССтанкоСбыт.

## Эталон
Файл: `RSS-Catalog-PromPortal-3000.xlsx`, строки 2–20 (19 станков).  
Эти описания написаны по ТКП‑1Н65‑эталону и содержат:
- 7 блоков (Назначение → Возможности → Конструкция → ТТХ → Комплектация → Применение → Подбор).
- 3500–6500 символов.
- Инженерный тон, без HTML‑мусора, без брендов ЧПУ.
- SERVICES_BLOCK внизу, неизменный.

## Критерии проверки

1. **Структура** (7 блоков присутствуют, логический порядок сохранён).  
2. **Полнота ТТХ** (диаметры, обороты, подачи, РМЦ, мощность, габариты, масса — если есть источники).  
3. **Комплектация** (стандарт + опции описаны отдельно).  
4. **HTML‑мусор** (нет `&Oslash;`, `&mdash;`, `&nbsp;` и т.п.).  
5. **SERVICES_BLOCK** (присутствует, текст совпадает с эталонным, не дублирован).  
6. **Тон** (инженерный, без рекламы, без брендов Siemens/FANUC/Heidenhain).  
7. **Длина** (не меньше 1500 символов для деталей, 3500+ для станков).

## Формат вывода

Для каждой проверенной строки:
- ✅ / ⚠️ / ❌ по каждому критерию.  
- Краткий комментарий, если не ОК (например: «Нет блока Комплектация», «HTML‑сущность `&mdash;` в строке 3»).  
- Общая оценка: OK / NEEDS_FIX / CRITICAL.
