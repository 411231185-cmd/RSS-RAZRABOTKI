# Навык: generate_description_prompt

## Назначение
Генерация готового промта для Claude Excel / Content‑Factory для создания или обновления описаний товаров.

## Входные данные

1. **Файл каталога** (например, `RSS-Catalog-PromPortal-3000.xlsx`).  
2. **Диапазон строк** (например, строки 21–50).  
3. **Тип задачи**:  
   - `audit` — только проверка (найти пустые описания, битые фото, HTML‑мусор).  
   - `generate` — генерация новых описаний по эталону (для пустых или кривых описаний).  
   - `import_ready` — подготовка файла для загрузки на площадку (PromPortal, Тильда и т.п.).  
4. **Источники данных** (ТКП, XLS ТТХ, RZN‑каша, Directus CSV и т.п.).  
5. **Эталон** (строки 2–20 из того же файла).

## Формат промта

Промт должен быть **конкретным и исполняемым**, с явным указанием:
- Структуры файла (какие колонки).  
- Где брать данные (файлы‑источники).  
- Формата описания (7 блоков, ~3500–6500 символов, SERVICES_BLOCK внизу).  
- Что НЕ трогать (SERVICES_BLOCK, коды товаров, ссылки).  
- Служебных колонок (если нужны флаги `DESCRIPTION_ISSUE`, `PHOTO_ISSUE`).

## Пример вывода

**Задача**: сгенерировать описания для токарно‑винторезных станков в строках 21–30 файла `RSS-Catalog-PromPortal-3000.xlsx`.

**Промт для Claude Excel**:

```plaintext
Рабочий файл: `RSS-Catalog-PromPortal-3000.xlsx`, основной лист.

Структура:  
- A = Наименование  
- B = Категория на портале  
- C = Товарная группа  
- D = Описание (верх — техописание, низ — SERVICES_BLOCK)

Задача: генерация описаний для строк 21–30.

Эталон: строки 2–20 этого же файла (станки 1Н65, 1М63Н, 16К40 и др.).  
Образец структуры: 7 блоков (Назначение → Возможности → Конструкция → ТТХ → Комплектация → Применение → Подбор).

Источники ТТХ:  
- `Технические_характеристики_станков.xls` (для токарно‑винторезных).  
- `RZN-ОПИСАНИЕ-каша.md` (для трубонарезных).  
- `ТКП-1Н65-ДИП500-ЭТАЛОН.md` (эталонная структура).

Правила:  
1. Не менять SERVICES_BLOCK в низу колонки D.  
2. Убрать все HTML‑сущности (`&Oslash;`, `&mdash;`, `&nbsp;`).  
3. Не упоминать бренды систем ЧПУ (Siemens, FANUC, Heidenhain).  
4. Описание: 3500–6500 символов, инженерный тон.  
5. Если ТТХ неполные — написать то, что есть, но структуру соблюсти.

Служебные колонки (добавь):  
- E = DESCRIPTION_ISSUE (флаг: `OK` / `EMPTY` / `HTML_MESS` / `TOO_SHORT`).  
- F = NOTES (комментарий: какие ТТХ отсутствуют).

Начни с модели в строке 21, далее по порядку до строки 30.
```
